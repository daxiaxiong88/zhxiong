# 我的摄影作品集（使用说明）

本项目是一个静态网页，纯展示个人摄影作品：不裁剪、不统一尺寸、无筛选、无点击放大。页面使用 Tailwind CSS（CDN 方式），并使用少量 JavaScript 仅用于“瀑布流布局”的列高平衡（无任何交互）。

## 目录结构

- `index.html`：主页面，包含分类筛选与点击放大功能。
- `Images/`：图片资源目录（请将照片按类别放入子文件夹）。
  - `Images/建筑/`
  - `Images/自然/`
  - `Images/人像/`
  - `Images/其他/`

## 分类与路径映射

页面中的按钮与图片项使用英文的 `data-category`，与中文目录的对应关系如下：

- `architecture` → `Images/建筑/`
- `nature` → `Images/自然/`
- `portrait` → `Images/人像/`
- `other` → `Images/其他/`

> 说明：目录名采用中文（与您的现有文件夹一致），HTML 中路径使用正斜杠 `/`。

## 如何新增或替换照片

1. 将照片文件复制到对应的 `Images/分类` 文件夹（如：`Images/建筑/_DSC3718.jpg`）。
2. 在 `index.html` 中定位到对应分类的 `section` 区块（例如“建筑”部分的 `<section>`），在其内部的图片网格容器（`<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6">`）按下面示例添加图片项：

```html
<div class="gallery-item" data-category="architecture">
  <img src="Images/建筑/_DSC3718.jpg"
       alt="建筑 _DSC3718"
       class="block max-w-full h-auto rounded-lg shadow-md mx-auto">
</div>
```

- 图片以原始比例完整展示：不设置固定高度、不使用 `object-cover`，避免裁剪。
- 网格保持响应式三列布局（手机一列、平板两列、桌面三列），各图片高度可不同。
- 若某类别不足 6 张，可选择保留占位卡片，也可不填充；不影响页面展示。

## 布局说明（Masonry）

- 页面为“瀑布流”布局：根据每张图片的真实高度自动分配到 1/2/3 列，使整体更均匀。
- 列数规则：手机 1 列（<640px）、平板 2 列（≥640px）、桌面 3 列（≥1024px）。
- 该分配通过少量 JavaScript 完成，不包含任何点击或筛选交互。

## 本地预览方式

- 方式一：直接双击打开 `index.html`。
- 方式二：启动本地服务器（推荐）：

```powershell
python -m http.server 8000
```

然后在浏览器访问：`http://localhost:8000/`

## 功能说明

- 按分类分区依次展示，不提供筛选或点击放大交互。
- 保持图片原始比例与完整内容，不裁剪或统一尺寸。

## 常见问题

- 路径包含中文是否可用？可以。HTML 中路径使用 `/`，浏览器可识别。
- 图片太大加载慢？建议为缩略图做压缩，并考虑添加 `loading="lazy"`。
- 图片显示比例不一致？这是预期设计：不裁剪、不统一尺寸，各图片高度将随原始比例而变化。
- 图片不显示？检查文件是否存在、路径是否一致（注意大小写与文件名）。

## 后续改进建议

- 为缩略图增加懒加载（`loading="lazy"`）与更小尺寸文件，加快首屏速度。
- 为每张照片增加标题/拍摄信息的文字说明。
- 通过脚本自动扫描 `Images` 目录并生成图片网格，减少手动重复工作。
